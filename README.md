# Dose-o-clock

Dose-o-clock is a local-only dosage timer built with Vue 3, TypeScript, Vite, and `vite-plugin-pwa`. It presents a dot-ring timer, lets you start dosage sessions with configurable values, keeps a short local history, and can be installed as a portrait-oriented PWA.

The app does not use a backend. Session data, history, and settings are stored in the browser's `localStorage`.

## How the Project Works

The application starts in `src/main.ts`, mounts `src/App.vue`, and renders the main timer experience through `src/components/MainView.vue`.

Core pieces:

- `src/components/` contains the Vue UI: the timer ring, bottom actions, new session panel, history panel, settings panel, and shared panel shell.
- `src/store/useTimerStore.ts` owns the reactive application state and actions for starting sessions, ending previous sessions, deleting history, resetting data, and saving settings.
- `src/store/storage.ts` loads and saves persisted state from `localStorage`.
- `src/domain/TimerSession.ts` models timer sessions, elapsed time, automatic four-hour stopping, and automatic 24-hour reset.
- `src/domain/dosage.ts` defines dosage ranges, supported increments, snapping logic, and display formatting.
- `src/pwa.ts` registers the service worker generated by `vite-plugin-pwa`.
- `vite.config.ts` configures Vue, TypeScript decorator support, the PWA manifest/workbox settings, aliases, HTTPS dev serving, and Vitest.

Timer behavior:

- A new session records a dosage value, start time, and optional earlier offset.
- Starting a new session automatically ends the previous active session and moves it into history.
- Active elapsed time is capped at four hours.
- Active sessions automatically stop once they reach four hours.
- Session data is cleared when the active session is at least 24 hours old.
- Settings and history persist locally in the browser only.

## Requirements

- Node.js
- npm

Dependencies are managed with `package-lock.json`, so use `npm install` to install the exact dependency tree.

## Install

```bash
npm install
```

## Run Locally

```bash
npm run dev
```

The Vite dev server is configured to run on:

```text
https://localhost:5175
```

The server uses `tomas.houseoftovig.com.pem` as both the HTTPS key and certificate. Because this is a local certificate, your browser may ask you to trust or accept it before loading the app.

## Build

```bash
npm run build
```

The build script runs TypeScript checking first, then creates the production bundle:

```bash
vue-tsc --noEmit && vite build
```

Build output is written to `dist/`.

## Preview the Production Build

```bash
npm run preview
```

The preview server uses the same HTTPS host and port configuration as the dev server:

```text
https://localhost:5175
```

## Test and Lint

Run the unit tests:

```bash
npm test
```

Run ESLint:

```bash
npm run lint
```

Tests use Vitest with the `jsdom` environment. Current tests cover domain timer behavior, dosage calculations, and local storage loading/persistence behavior.

## PWA Notes

The app is configured as a PWA through `vite-plugin-pwa`.

- The manifest name is `Dose-o-clock`.
- The app uses standalone display mode and portrait orientation.
- Static PWA assets include `public/icons/icon.svg`.
- The generated service worker precaches built JS, CSS, HTML, SVG, PNG, and ICO files.
- Navigation falls back to `/index.html`, which allows normal single-page app routing behavior.
- The service worker is registered immediately from `src/pwa.ts`.
